# Paper Assistant Lite v3.4.0 — 学术论文全流程高级助手

> **为什么做这个 skill**
>
> 当你第三次告诉 AI "这段读起来像机器写的，帮我改自然一点"，却拿到一份只是把"显著提升"换成"有了明显进步"的换词稿时——问题不在你，在于 AI 缺乏一个真正懂教育学论文的"内部审稿人"。
>
> 学术写作的 prompt 工程正在成为一种隐性资源：有经验的研究者有一套自己的润色模板和自检清单，而大多数人还在从零摸索、反复调教。这个 skill 的目标很简单：**把学术论文写作中经过验证的方法论、模板和质量检查机制，打包成一个 AI 可以稳定执行的技能包。让你不用在 prompt 调试上浪费时间，把精力留给真正的思考和写作。**

## 概述

v3.4 定位：**面向国内硕士研究生的全流程学术写作助手**（示例以教育学为主，方法论适用于所有人文社科及部分理工科领域）。在 v3.3 基础上重构所有 Prompt 模板（参考 awesome-ai-research-writing 项目设计哲学），引入修改阈值系统、三层 Role 构造、定制化 Execution Protocol、条件分支输出格式。

核心升级：
- **学位论文写作模式（P0）**：国内硕士论文完整结构（绪论→文献综述→理论基础→研究设计→研究结果→讨论→结语），每章配写作指导和检查清单
- **中文文献搜索增强（P1）**：知网/万方/维普/百度学术/期刊官网专线路，适配中文期刊DOI注册率低的现实
- **轻量综述模式（O3）**：不强制走完整 PRISMA 流程，硕士论文文献综述常用轻量模式
- **开题报告模板（O1）**：国内硕士开题标准结构
- **国内硕士培养流程编排（O6）**：开题→中期→预答辩→答辩，替代国际期刊投稿流程
- **文献池跨会话持久化（O5）**：长论文跨会话写作不丢文献
- **研究工具设计指南（A3）**：问卷编制全流程（成熟量表/修订/自编三条路线）、访谈提纲设计、观察记录表设计
- **伦理审查指南（A5）**：伦理申请表框架、知情同意书模板、常见审查问题应对
- **AIGC 率降低七规则（C1）**：句长落差、引述句式多样化、段落结构反对称、研究者声音、领域内部用语、具体化原则、不完美允许
- **学术图表生成（C2）**：完整图表工具链，含描述统计表/相关矩阵/条形图/箱线图/散点图/三线表等7大类型、Python代码即用、论文图表质量标准、图表与章节对应关系
- **去除草稿迭代**：不再内置 R&R 修改矩阵和迭代修订流程，一次产出到位

**核心哲学不变**：引用来源锁定——先搜索→入池→引用，零伪造。

## 执行优先级与最低遵循清单

> 本 skill 内容较长（2000+ 行），在上下文有限时，AI 可能无法同时遵循所有指令。以下按优先级排序，若只能遵循 N 条，优先遵循前 N 条。

| 优先级 | 规则 | 适用场景 | 违反后果 |
|--------|------|---------|---------|
| P0 | 引用来源锁定——先搜索验证入池引用，零伪造 | 涉及文献的任何环节 | 论文废掉 |
| P0 | 正文内容必须基于真实文献来写，不编造 | 写作所有章节 | 论文废掉 |
| P1 | 统计方法选择必须先检验前提假设（正态/方差齐/线性等） | 数据分析 | 方法选错结论无效 |
| P1 | 研究设计中的工具必须有信效度说明 | 第4章+开题报告 | 方法学评审不通过 |
| P2 | AIGC 七规则 | 写作全部正文 | AIGC 检测率偏高 |
| P2 | 三线表格式 + 效应量 + 置信区间报告 | 结果章节 | 格式不规范 |
| P3 | Role 三层构造 + 修改阈值（宁缺毋滥） | 使用附录A模板时 | 文本质量下降 |
| P3 | 反方检验 + 替代解释 | 讨论章节 | 论证深度不足 |

使用建议：新对话开始时告知 AI "请遵循 P0 和 P1 规则"即可启动。上下文紧张时放弃 P3，优先保住 P0-P1。附录A模板是即用即加载的——用户说润色 AI 才调用润色模板。

## 引用来源锁定系统（所有模块通用，最高优先级）

> ⚠️ 本技能最根本的铁律。引用在进入论文之前就必须是真实的。不存在"事后审计"，不存在"待验证"。

### 核心原则

```
搜索 → 验证 → 加入文献池 → 从池中引用
```

### 已验证文献池

所有模块共享。每条文献包含：

```
条目ID | 标题 | 作者 | 年份 | 期刊/出版社 | DOI(如有) | 验证来源 | 验证时间 | 摘要/关键发现 | 被引次数
```

### 引用入池流程

1. **搜索**：用 WebSearch / Semantic Scholar / Crossref / 中文数据库搜索相关文献
2. **确认**：至少一个权威来源确认文献真实存在
3. **入池**：提取完整元数据，写入文献池
4. **引用**：从池中取出，格式化后插入正文

**任何跳过步骤1-3直接写引用的行为都是违规。**

### 搜索不到的处理

- 如实告知用户搜索了哪些渠道、用了哪些关键词、均未找到
- **不将该文献写入论文**，不创建"待验证"标记
- 建议更换关键词或寻找替代文献

### 用户提供文献

用户提供的文献仍需搜索验证。验证通过→入池→引用；验证失败→告知用户，不在论文中使用。

### 跨会话文献池持久化（O5）

长论文写作通常跨多个会话。文献池通过文件持久化：

1. **自动保存**：每次文献入池操作后，将完整文献池写入 `文献池_<主题>.md` 文件
2. **会话启动加载**：新会话开始时，检查工作目录是否已有 `文献池_*.md` 文件，如有则读取并恢复
3. **增量更新**：新文献追加到已有池中，不覆盖
4. **去重**：加载时按标题+作者+年份去重

文献池文件格式：

```markdown
# 文献池 — [主题] — 最后更新: YYYY-MM-DD

## R001
- 标题: xxx
- 作者: xxx
- 年份: xxx
- 期刊: xxx
- 卷期页码: xxx
- DOI: xxx
- 验证来源: Crossref / Semantic Scholar / 期刊官网 / WebSearch
- 关键发现: xxx
- 被引次数: xxx
```

### 用户上传文献研读与入池

用户可以将自己收集的 PDF 论文、学位论文等文献直接发送给助手进行深度研读，而非仅依赖网络搜索。

适用场景：导师指定的必读文献、已下载但尚未精读的核心论文、同行分享的未发表手稿或预印本、数据库中下载的学位论文。

研读流程：

1. 用户上传文献（PDF 文件或截图）→ 助手使用 Read 工具读取 PDF 内容 → 自动提取元数据

2. 结构化提取（对每篇文献输出）：基本信息（标题/作者/年份/期刊/DOI）、研究问题、研究方法（设计类型+样本量+工具+分析方法）、核心发现（3-5点最重要的结果带数据）、理论框架、局限性（作者自认+读者视角）、与用户研究的关联（可引用到哪一章/支撑哪个论点）、关键引文（3-5句可引用原文需标注页码）

3. 入池（自动完成）：提取的元数据 + 结构化笔记写入文献池文件，与已有文献池去重合并，分配文献池条目ID

4. 交叉引用分析（批量上传多篇后）：高频被引作者/文献识别、研究脉络追踪（A引B，B引C……）、争议点识别（哪些结论不同文献互相矛盾）、研究缺口（多篇文献共同指出的未来研究方向）

批量上传后的知识库构建（完成5+篇文献研读后输出）：

1. 核心争论地图：哪些问题学术界有共识？哪些有分歧？你的研究可以介入哪个争论？
2. 方法学清单：这个领域最常用的3种研究方法，各方法的典型样本量和工具
3. 理论脉络图：主要的理论阵营及其代表人物，你的研究站在哪个理论阵营？
4. 研究缺口清单：至少3篇文献共同指出的缺口，等级划分：高优先（方法学缺口）/ 中优先（人群缺口）/ 低优先（场景缺口）

文献研读笔记模板（每篇上传文献产出一份）：

```
[R编号] — [作者] ([年份]) — [标题]
---
基本信息：期刊/来源/DOI/被引次数
研究问题：xxx
方法：设计=N=xxx, 来源=xxx, 工具=xxx, 分析=xxx
核心发现：1. xxx (p<.05, d=0.xx) 2. xxx
理论框架：xxx
局限：xxx
对用户研究的支撑：第X章用于支撑[某个论点]，可引用表述"xxx"（p.xx）
与池内文献的关系：与[Rxxx]结论一致，与[Rxxx]存在分歧（分歧点xxx）
```

## 模块一：深度文献研究

### 模式选择

在阶段1结束前，与用户确认使用哪种综述模式：

| 维度 | 系统综述模式 | 轻量综述模式 |
|------|------------|------------|
| 适用场景 | 独立系统综述论文、Meta分析、博士论文文献综述 | 硕士论文文献综述章、开题报告文献部分、快速了解领域 |
| 检索规模 | 多数据库、系统化关键词矩阵 | 2-4个关键词组合、1-2个数据库 |
| 筛选流程 | 完整 PRISMA 2020 四阶段 | 读标题+摘要筛选，无流程图 |
| 质量评估 | ROBIS/CASP + GRADE 证据评级 | 简要评估，不要求正式评级 |
| 出版偏倚 | 检测 | 不要求 |
| 输出 | PRISMA流程图 + 完整审计 + GRADE评级表 | 主题式综述报告 + 文献池 |

### 工作流

#### 阶段1：需求澄清（苏格拉底式递进提问）

1. **PICO/PICo 框架**：
   - P（Population）：研究对象
   - I（Intervention）：干预/关注点
   - C（Comparison）：对比条件（轻量模式可省略）
   - O（Outcome）：结局指标

2. **时空约束**：时间范围？中英文偏好？期刊级别要求？

3. **方法学聚焦**：定量/定性/混合方法？

4. **用途锚定**：学位论文综述章 / 开题报告 / 独立综述 / 期刊论文引言？

5. **确认综述模式**：系统综述 or 轻量综述？

#### 阶段2：多源系统检索

**中文文献检索路径（P1增强）**：

```
中文检索通道A: WebSearch
  ├── "<中文核心词1> <中文核心词2>"（通用搜索）
  ├── "<同义词变体> <研究方法词>"
  ├── "<核心概念> <研究对象> <干预>"（三词组合）
  └── 记录: N_A = ___

中文检索通道B: 知网/万方/维普定向搜索
  ├── site:cnki.net <关键词>（或 WebSearch "知网 <关键词>"）
  ├── WebSearch "万方数据 <关键词>"
  ├── WebSearch "维普 <关键词>"
  └── 记录: N_B = ___

中文检索通道C: 百度学术 + 期刊官网
  ├── WebSearch "百度学术 <关键词>"
  ├── WebSearch "<期刊名> <关键词>"（锁定核心期刊）
  └── 记录: N_C = ___
```

**英文文献检索路径**：

```
英文检索通道D: Semantic Scholar API
  ├── query="<English keyword 1> <English keyword 2>"
  ├── fields=title,authors,year,abstract,externalIds,citationCount,journal
  └── 记录: N_D = ___

英文检索通道E: Crossref API
  ├── 按关键词检索元数据
  └── 记录: N_E = ___
```

**关键词设计矩阵**（以下以教育学论文"生成式AI对自闭症儿童学习动机的影响"为例演示方法，你的论文替换为对应领域的关键词）：

| 维度 | 中文关键词 | 英文关键词 |
|------|-----------|-----------|
| 核心概念1 | 生成式AI/人工智能/大语言模型/ChatGPT | generative AI/LLM/ChatGPT |
| 核心概念2 | 自闭症/孤独症/ASD/谱系障碍 | autism/ASD/autism spectrum |
| 核心概念3 | 学习动机/学习兴趣/参与度 | learning motivation/engagement |
| 场景 | 随班就读/融合教育/特殊教育 | inclusive education/special education |
> 领域适配：矩阵的维度结构不变（核心概念1/2/3+场景），替换的是关键词内容。医疗论文可能是"二甲双胍/胰岛素抵抗/2型糖尿病/基层医疗"；管理学可能是"远程办公/员工满意度/组织承诺/互联网企业"。

**中文文献验证的特殊性**：

中文期刊 DOI 注册率低，验证路径与英文不同：

```
有DOI → Crossref API 精确匹配（标准路径）
无DOI → WebSearch "<期刊名> <年份> <作者> <标题关键词>"
        → 找到期刊官网或知网/万方页面
        → WebFetch 确认元数据（标题、作者、卷期页码）
        → 入池（验证来源标注为"期刊官网"或"知网"）
```

轻量模式：每个关键词组合 2-3 组即可。系统综述模式：建议 6-8 组。

**教育学中文核心期刊白名单（检索优先锁定）**：

#### 阶段3：筛选（分模式）

**系统综述模式——PRISMA 2020 四阶段**：

| 阶段 | 操作 | 记录 |
|------|------|------|
| Identification | 所有检索命中去重 | N₁ |
| Screening | 读标题+摘要排除 | N₂ |
| Eligibility | WebFetch全文/摘要页判断 | N₃ |
| Included | 最终纳入 | N₄ |

排除原因分类：重复/非同行评审/主题不相关/方法学不可评估/无法获取信息/语言不可读。

**轻量综述模式——简化筛选**：
1. 浏览所有检索结果标题
2. 筛选摘要相关文献（记录筛选标准）
3. WebFetch 获取摘要/全文确认相关性
4. 记录最终纳入数量和排除数量

**不要求** PRISMA 流程图、排除原因分类表。

#### 阶段4：文献入池与数据提取（两种模式共用）

1. **搜索确认**：按中文/英文验证路径确认文献真实存在
2. **提取元数据入池**（格式见"已验证文献池"节）
3. **质量评分**（每条文献入池时按以下维度打分）：
   | 维度 | 满分 | 评分标准 |
   |------|------|---------|
   | 期刊级别 | 3 | CSSCI/SSCI=3 / 北大核心=2 / 普通=1 / 非学术=0 |
   | 被引次数 | 2 | ≥50次=2 / 10-49=1 / <10=0 |
   | 时效性 | 2 | 近3年=2 / 3-5年=1 / >5年=0 |
   | 方法学质量 | 2 | 实证研究=2 / 理论分析=1 / 经验总结=0 |
   | 相关性 | 1 | 高度相关=1 / 部分相关=0 |

#### 阶段5-7：质量评估（仅系统综述模式）

轻量综述模式跳过这三个阶段。

**阶段5：方法学质量评估**
- 定量研究 → ROBIS / Cochrane RoB 2
- 质性研究 → CASP 质性研究清单

**阶段6：GRADE 证据质量评级**

| 证据质量 | 符号 | 含义 |
|---------|------|------|
| 高 | ⊕⊕⊕⊕ | 进一步研究不太可能改变效应估计 |
| 中 | ⊕⊕⊕⊖ | 进一步研究可能改变效应估计 |
| 低 | ⊕⊕⊖⊖ | 进一步研究很可能改变效应估计 |
| 极低 | ⊕⊖⊖⊖ | 效应估计非常不确定 |

降级因素：偏倚风险、不一致性、间接性、不精确性、出版偏倚（各 -1~-2）

**阶段7：出版偏倚检测**
≥10篇时：描述偏倚方向、预注册文化、未发表研究子领域。

#### 阶段8：综合分析报告

**系统综述模式输出**：
1. PRISMA 2020 流程图（带数字和排除原因）
2. 文献特征矩阵：作者/年份/国家/设计/样本量/干预/主要发现/效应量/质量评分
3. 主题聚类分析（3-5个核心主题）
4. 方法学质量概览 + GRADE评级
5. 出版偏倚评估
6. 研究缺口地图
7. 可操作建议
8. 参考文献（全部来自文献池）

**轻量综述模式输出**：
1. 检索概述（用了哪些关键词、哪些数据库、各命中多少）
2. 纳入文献列表（表格：作者/年份/研究主题/主要发现）
3. 主题综合（按主题分段叙述，每段整合多篇文献的观点）
4. 研究缺口识别（哪些方面研究不足）
5. 文献池文件

### 学位论文模式（P0+O2）

#### 阶段1：风格校准

确认：
1. 学位类型（硕士/博士）、专业方向
2. 学校格式要求（封面、字体、行距、页眉页脚等——如用户有模板文件，读取并遵循）
3. 目标字数（硕士常见 3-5万字，以用户为准）
4. 引文格式（国内硕士论文一般用 GB/T 7714）
5. 语言（中文为主 / 英文为主 / 双语）
6. 研究问题与核心概念（1-3个研究问题）

#### 研究问题锐化器（阶段1.5，进入正文写作前必须完成）

研究问题的质量决定了整篇论文的天花板。在正式写作前，对每个研究问题进行5项锐化检验：

```
对研究问题"生成式AI能否提升随班就读自闭症儿童的学习动机？"的锐化：

1. 可回答性检验：
   这个问题的答案可以通过你手头的方法和数据得出吗？
   → "能否"过于二元。改为"在何种条件下、以何种机制产生影响"更可回答。

2. 可证伪性检验：
   如果结果不支持你的假设，结论是什么？
   → 如果AI组和对照组无显著差异，论文仍有意义——说明简单引入AI是不够的。

3. 创新性检验：
   前人已经回答到什么程度？你的问题在哪个维度上是新的？
   → 已有AI+普教的研究，但AI+自闭症+随班就读场景的研究极少。

4. 重要性检验：
   这个问题对实践者（教师/家长/管理者）意味着什么？
   → 如果发现AI有效，随班就读教师就多了一个低成本的支持工具。

5. 聚焦度检验：
   这个问题能在3-5万字的硕士论文范围内回答清楚吗？
   → 如果涉及的因素太多（AI类型×自闭症程度×年级×学科），需要聚焦。
```

**输出**：锐化后的研究问题 + 每个问题对应的假设/子问题 + 方法路线图。

#### 阶段2：论文结构选择

提供三种常见结构，用户选择或自定义：

**结构A（七章制，最常见）**：

```
前置：封面 → 中文摘要(300-500字) → 英文摘要 → 目录 → 图表清单
主体：
  第1章 绪论（~3000字）
    1.1 研究背景
    1.2 研究问题的提出
    1.3 研究目的与意义
    1.4 核心概念界定
    1.5 研究设计概述
    1.6 论文结构安排
  第2章 文献综述（~5000字）
    2.1 [核心概念1]的研究现状
    2.2 [核心概念2]的研究现状
    2.3 [核心概念3]的研究现状
    2.4 已有研究的不足与本研究的切入点
  第3章 理论基础（~3000字）
    3.1 [理论A]及其对本研究的启示
    3.2 [理论B]及其适用性分析
    3.3 本研究的理论框架
  第4章 研究设计（~4000字）
    4.1 研究问题与研究假设
    4.2 研究对象
    4.3 研究方法
    4.4 研究工具
    4.5 数据收集程序
    4.6 数据分析方案
    4.7 研究伦理
  第5章 研究结果（~6000字）
    5.1 [研究问题1]的结果
    5.2 [研究问题2]的结果
    5.3 [研究问题3]的结果（如适用）
  第6章 讨论（~4000字）
    6.1 主要发现讨论
    6.2 与已有研究的对话
    6.3 研究局限
    6.4 实践建议
  第7章 结语（~1500字）
    7.1 研究结论
    7.2 研究贡献
    7.3 研究展望
后置：参考文献 → 附录 → 致谢 → 攻读学位期间发表的学术论文
```

**结构B（六章制，理论基础并入综述）**：
绪论 → 文献综述与理论基础（合章）→ 研究设计 → 研究结果 → 讨论 → 结论

**结构C（五章制，紧凑型）**：
绪论 → 文献综述 → 研究方法 → 研究结果与讨论 → 结论

#### 阶段3：论证逻辑骨架

在写正文前构建骨架：

```
核心研究问题: _________________________________

子问题1 → 支撑文献 [R001][R002][R003] → 回答逻辑
    ↕
子问题2 → 支撑文献 [R004][R005] → 回答逻辑
    ↕
子问题3 → 支撑文献 [R006][R007][R008] → 回答逻辑
```

检查：
- 每个子问题有 ≥2 条文献支撑？
- 子问题间有逻辑递进？
- 文献全部来自文献池？

**反方检验（Devil's Advocate Check）**：

在骨架确认后、正文写作前，对每个子问题执行反方检验：

```
子问题1：AI对话是否能提升自闭症儿童的学习动机？
  ├── 已有文献支持方: [R001] [R003] [R005]  ← 入池时确认
  ├── 反方最强论点: "儿童对AI对话的新颖效应（novelty effect）消退后动机回落"
  │   └── 支持反方的文献: [R012]（该研究发现干预后3个月效果减半）
  ├── 你的回应: 本研究设计了延迟后测（干预结束3个月后），可检验新颖效应假说
  └── 反方论点是否已消解？☑ 已通过研究设计消解

子问题2：AI干预对不同年级的自闭症儿童效果是否一致？
  ├── 反方最强论点: "低年级儿童操作AI设备有困难，差异可能是技术障碍而非动机问题"
  ├── 你的回应: 研究中如果用语音交互而非文本输入，可以降低技术门槛
  └── 反方论点是否已消解？⚠ 未完全消解，需要在讨论中作为局限性说明
```

**反方检验的输出**：
- 每个子问题配 1 个最强反方论点（基于文献池）
- 每个反方论点配 1 句回应（研究设计消解 / 讨论中承认并说明 / 需要在未来研究中检验）
- 至少 1 个反方论点标记为"未完全消解"→写入第6章局限部分

#### 阶段4：分章写作指导与检查清单

**第1章 绪论 检查清单**：
- [ ] 研究背景从宏观到微观（领域→具体问题→本研究）
- [ ] 研究问题清晰、可操作、可回答
- [ ] 研究意义区分理论意义和实践意义，不夸大
- [ ] 核心概念界定有文献支撑，不是自己编的定义
- [ ] 研究设计概述与第4章对应

**第2章 文献综述 检查清单**：
- [ ] 按主题组织（不是按作者罗列）
- [ ] 每个主题有综合叙述（"张三(2020)发现X，李四(2021)进一步验证了Y" 而非 "张三说了X。李四说了Y。"）
- [ ] 末尾有研究述评：已有研究的共识、不足、本研究的切入点
- [ ] 所有引用来自文献池
- [ ] 经典文献与最新文献兼顾（近5年文献占比 >50%）

**第3章 理论基础 检查清单**：
- [ ] 与文献综述的区别清晰：综述是"别人做了什么"，理论章是"我基于什么理论框架"
- [ ] 每个理论有原始文献引用（引用提出者，不引用二手转述）
- [ ] 明确说明该理论与本研究的关联（不是罗列理论）
- [ ] 理论框架图清晰呈现各理论间关系及其对研究问题的指导

**第4章 研究设计 检查清单**：
- [ ] 研究方法选择有理由（为什么选这个方法？与其他方法比较？）
- [ ] 研究对象描述完整（人数、来源、入选/排除标准）
- [ ] 研究工具信效度报告完整（引用原始开发文献+本研究的信效度数据）
- [ ] 数据收集程序可重复
- [ ] 数据分析方案与研究假设一一对应
- [ ] 伦理声明完备（批准机构+编号，如已获得）

#### 研究工具设计指南（A3）

教育学硕士论文常用的研究工具包括问卷/量表、访谈提纲、观察记录表、测验等。以下按工具类型给出设计流程。

**问卷/量表设计流程**：

```
步骤1: 明确测量构念
  ├── 要测什么？（如：学习动机、融合教育态度、教师自我效能感）
  ├── 维度划分（如：内在动机/外在动机/任务价值/自我效能感）
  └── 从文献中提取维度依据（文献池中应有支撑）

步骤2: 选择编制路线
  ├── 路线A: 直接使用成熟量表（推荐，省时省力，信效度已有数据）
  │   ├── 从文献中筛选：该量表在类似人群中使用过
  │   ├── 获取使用授权（部分量表需申请）
  │   └── 论文中说明：引用原始量表文献 + 引入本研究时的修订说明
  ├── 路线B: 修订成熟量表（跨人群/跨情境）
  │   ├── 保留核心维度，调整/删除/新增题目
  │   ├── 删除后需重新做信效度检验
  │   └── 论文中说明：修订了什么、为什么修订、修订后信效度
  └── 路线C: 自编量表（仅在无合适成熟量表时使用）
      ├── 遵循完整编制流程（见下方自编量表六步法）
      └── 论文中必须完整报告编制过程和信效度数据

步骤3: 题目编写
  ├── 题目数量：每个维度 3-5 题，总题量 20-40 题
  ├── 题型：Likert 5点/6点量表为主（1=完全不符合 → 5=完全符合）
  ├── 正向题与反向题比例约 4:1
  ├── 题目表述：避免双重否定、避免引导性、避免专业术语
  └── 每道题只测一个方面（避免"双重负载"）

步骤4: 专家评审（内容效度）
  ├── 邀请 3-5 位领域专家评审题目
  ├── 评审维度：相关性、清晰度、代表性
  ├── 计算 I-CVI（条目水平内容效度指数）：≥0.78 为优秀
  ├── 计算 S-CVI（量表水平内容效度指数）：≥0.80 为合格
  └── 根据专家意见修改/删除题目

步骤5: 预测试
  ├── 小样本（30-50人，与研究样本同质）
  ├── 目的：检查题目可理解性、完成时间、题目难度分布
  ├── 分析：题目均值、标准差、偏度、峰度
  └── 删除：均值极端（<2或>4.5）、标准差<0.5、偏度>2 的题目

步骤6: 正式施测与信效度检验
  ├── 信度：
  │   ├── Cronbach's α（内部一致性）：总量表 ≥0.80，各维度 ≥0.70
  │   ├── 分半信度（如需要）
  │   └── 重测信度（如适用，间隔2-4周）：≥0.70
  ├── 效度：
  │   ├── 结构效度：探索性因子分析 EFA（KMO>0.7, Bartlett球形检验 p<0.05）
  │   ├── 因子累积方差解释率：≥50%
  │   ├── 题目因子载荷：≥0.50
  │   ├── 验证性因子分析 CFA（如样本量够）：CFI>0.9, RMSEA<0.08
  │   └── 效标关联效度（如有外部效标）
  └── 论文报告格式：
      - "本研究采用张三(2018)编制的XX量表，共X个维度X道题..."
      - "修订说明：删除了X道题，新增了X道题..."
      - "信度检验：总量表Cronbach's α=0.89，各维度α介于0.75-0.88之间"
      - "效度检验：EFA提取X个因子，累积方差解释率62.3%，各题目因子载荷0.52-0.85"
```

**自编量表六步法（仅路线C需要完整走）**：

```
1. 文献回顾与构念定义 → 2. 初始题库建立（文献+访谈补充）→
3. 专家评审（内容效度）→ 4. 预测试（项目分析）→
5. 正式施测（EFA/CFA）→ 6. 形成正式量表
```

**访谈提纲设计流程**：

```
步骤1: 确定访谈类型
  ├── 半结构化（最常用）：有核心问题框架，允许追问
  ├── 结构化：固定问题顺序，适用于多人对比
  └── 非结构化：仅确定主题方向，适用于探索性研究

步骤2: 拟定提纲框架
  ├── 开场热身（1-2题）：破冰，建立信任
  ├── 核心问题（5-8题）：围绕研究问题展开
  │   ├── 每个核心问题对应一个子问题/主题
  │   ├── 开放式提问："请您谈谈…""您觉得…""能否举例说明…"
  │   └── 避免诱导："您是不是觉得…""难道不…"
  └── 结束语（1题）：还有其他想补充的吗？

步骤3: 专家评审
  ├── 请1-2位有质性研究经验的专家审阅提纲
  ├── 检查：问题是否有逻辑顺序、是否可能引发不适、是否遗漏关键主题
  └── 根据反馈修改

步骤4: 试访谈
  ├── 找1-2名与研究样本类似但不纳入正式研究的人试访谈
  ├── 检查：问题是否清晰、是否需要追问、时间是否合适
  └── 记录试访谈中发现的问题，修改提纲

步骤5: 定稿
  └── 论文中附访谈提纲（放附录）
```

**观察记录表设计**：

```
结构化观察表基本要素：
├── 基本信息：观察日期、时间、地点、观察者
├── 观察对象：编号/化名、基本信息
├── 观察情境：活动类型、环境描述
├── 观察维度（预先设定，与研究问题对应）：
│   ├── 每个维度 3-5 个观察指标
│   ├── 记录方式：频率记录 / 时长记录 / 等级评定
│   └── 评分标准清晰明确
└── 备注栏：记录预设指标之外的意外行为
```

**第5章 研究结果 检查清单**：
- [ ] 按研究问题组织，每个问题独立一节
- [ ] 先描述后推断，先总体后具体
- [ ] 定量：报告效应量 + 置信区间（不只p值）
- [ ] 定性：有原始数据引述（匿名化）
- [ ] 图表编号规范，表题在上、图题在下
- [ ] 缺失数据已说明处理方式
- [ ] 📊 图表使用 Python（matplotlib/seaborn）生成，符合论文分辨率与格式标准，详见**附录C**

**第6章 讨论 检查清单**：
- [ ] 第1段概括最重要的发现
- [ ] 每段讨论一个发现 → 与文献对比 → 解释机制
- [ ] **替代解释检查**：每个主要发现至少列出 1 个替代解释（如：是AI效果好，还是新鲜感消退前的新颖效应？）→ 分析哪种解释更有可能
- [ ] 局限性具体诚实（"本研究的样本来自某市三所小学，不能推广到农村地区"，不说"样本量小"），且每个局限必须配一句"这对结论意味着什么"的说明
- [ ] **反方回应**：与至少3篇入池文献的不同结论进行主动对话（不是绕开，而是解释差异）
- [ ] 实践建议具体可操作，面向目标读者（教师/管理者/研究者）

**第7章 结语 检查清单**：
- [ ] 简洁回扣研究问题
- [ ] 研究贡献不夸大
- [ ] 展望方向具体

#### 阶段5：全局质量审查

**可读性**：
- 句长有落差：同一段落内短句（≤15字）与长句（≥40字）交替出现，拒绝句长均匀分布
- 段落长度灵活：允许50字点睛段和300+字深入段，不要求统一区间（详见阶段5.5规则7）
- 术语全文一致

**引用平衡**：
- 自引 < 20%
- 中文 vs 英文比例合理
- 经典（>10年）vs 新文献（<5年）平衡
- 无引用空白章节

**过渡连贯**：
- 段间有逻辑桥梁
- 章节末预告下一章

#### 阶段5.3：出乎意料模式检测（讨论前必做）

> ⚠️ 很多硕士论文的讨论部分写成了"验证了假设"的报账式总结——因为缺乏对数据中意外模式的敏感度。

在进入讨论章写作前，用以下问题扫描分析结果：

```
1. 反直觉检测：
   有没有哪个结果是和你预期相反的？
   → 例：你预期AI干预会提升动机，但低年级组的动机反而下降了？
   → 这不一定是坏事——反直觉的结果往往是最有趣的部分

2. 异常子群体检测：
   在整体趋势之下，有没有某一小群人的模式完全不一样？
   → 例：男生和女生对AI的反应差异巨大？高焦虑和低焦虑学生差异？
   → 拆分子群体分析比整体均值更有深度

3. 故事断裂检测：
   你的三个子问题的结果之间，有没有逻辑上的断裂或矛盾？
   → 例：动机提升了，但成绩没变？参与度高了但焦虑也高了？
   → 断裂之处往往是讨论最有价值的起点

4. 文献冲突检测：
   你的发现中，有没有和某篇入池文献的结论直接冲突的？
   → 你的结论如果是"AI对自闭症儿童有效"，但文献池中有2篇说"效果有限"
   → 不是绕开，而是主动对话：为什么你和他们得出不同结论？方法差异？样本差异？
```

**输出**：以上四项扫描结果写入讨论章提纲。如果四项全无发现，则讨论章可能过于平铺直叙——考虑做更细粒度的子群体分析。

#### 阶段5.5：AIGC 率降低审查

> 以下规则中使用了教育学领域的示例来说明方法。如果你的论文是其他领域（医学、管理、工程……），核心方法不变——句长落差、引述多样、研究者声音——但具体的场景描述、领域用语、案例细节应替换为你所在领域的等价物。方法通用，内容来自你的领域。

> AIGC 检测系统的核心抓取两类特征：**低困惑度**（用词太标准、太可预测）和**低突发性**（句长太均匀、结构太工整）。以下规则强制执行，贯穿学位论文模式、期刊论文模式和开题报告模式的全部写作。

**规则1：句长有落差，拒绝均匀**

- ❌ 禁止：每句长度都在 25-35 字之间均匀分布
- ✅ 要求：同一段落内必须出现短句（≤15字）和长句（≥40字）
- 短句用于断言、强调、转折："这一点至关重要。""但问题在于。"
- 长句用于展开论证、列举、因果说明
- 段落首句可以是短句（开门见山）也可以是长句（铺设背景），但不要每段都一样

**规则2：引述句式多样化**

同一段落内禁止连续使用相同句式引用文献。从以下句式中交替选用：

| 句式类型 | 示例 |
|---------|------|
| 直接引述 | "张三(2020)指出，融合教育环境下的学习动机显著高于隔离环境。" |
| 间接引述 | "在张三(2020)的研究中，融合班级学生的课堂参与度提升了23%。" |
| 批判性引述 | "张三(2020)虽然证实了X，但其样本仅限于城市学校，农村情境下的结论尚不明确。" |
| 整合式引述 | "多项研究（张三,2020;李四,2021;王五,2022）一致表明…" |
| 数据锚定引述 | "以张三(2020)在3所小学的实验为例，干预组前后测差异达到…" |
| 对比式引述 | "与张三(2020)的发现不同，李四(2021)在乡村学校的观察中并未发现类似趋势。" |
| 承前启后式引述 | "在前文梳理了X的基础上，张三(2020)进一步揭示了Y的机制。" |

**规则3：段落结构反对称**

- ❌ 禁止：每段都是"主题句→展开→总结"的标准三段式
- ✅ 要求：混合使用以下段落开头方式，相邻段落不重复：
  - 数据/案例开头（倒叙）："'你这孩子怎么老是不听话！'——这是笔者在随班听课中听到频率最高的一句话。"
  - 设问开头："生成式AI能否真正理解自闭症儿童的学习需求？答案远比想象中复杂。"
  - 文献分歧开头："关于X的影响，学界尚未达成共识。"
  - 直接断言开头："学习动机的下降不是突然发生的。"
  - 场景描写开头："走进三年二班的教室，小明依然坐在最后一排靠窗的位置。"
- 有些段落最后一句不给总结性收束，直接以数据、引用或疑问结束，留白给读者

**规则4：注入研究者的声音**

- 适当使用第一人称学术表达："笔者认为"、"在笔者的实地观察中"、"本研究与已有研究的分歧在于"
- 对文献要有判断和态度，不是纯客观转述：
  - "这一结论值得商榷，原因在于…"
  - "该研究最大的贡献在于首次将X引入Y领域"
  - "遗憾的是，这项研究未能控制Z变量"
  - "这组数据看似矛盾，实则揭示了…"
- 人类写论文有立场，不要"端水"

**规则5：使用领域内部用语**

教育学研究者内部有习惯用语和"行话"，适当使用可降低 AIGC 特征：

| 正式表达（易被标记） | 领域用语（更自然） |
|-------------------|-----------------|
| 随班就读质量不高 | 随班就读"随"而不"读" |
| 融合教育支持不足 | "随班就读"变成了"随班就坐" |
| 教师对自闭症儿童支持有限 | 普教老师不知道怎么接特教的孩子 |
| 特殊教育资源分配不均 | 乡镇学校配不起资源老师 |
| 家长参与度低 | 家长把孩子往学校一送就不管了 |
| 个别化教育计划执行困难 | IEP 写完就锁抽屉里了 |

⚠️ 注意：领域用语用于叙述和讨论部分，绪论、研究设计等正式章节仍保持学术书面语。
> 领域适配说明：上表中的"领域用语"是教育学示例。在医疗论文中，"这个病人依从性差"比"患者治疗依从性不足"更自然；在管理学中，"员工干着干着就走了"比"员工离职倾向显著"更像研究者写的话。关键不是照搬教育学的词，而是找到你所在领域研究者平时怎么说话——那些在办公室里、病房里、工地上的真实表达。

**规则6：具体化原则**

- ❌ 抽象："提升了学习动机"
- ✅ 具体："学生从原来整节课低头不语，到后来能主动举手回答两次问题"
- ❌ 抽象："教师融合教育素养不足"
- ✅ 具体："在笔者观察的12节课中，只有1位教师在教案中提及了对特殊需要学生的差异化安排"
- 数据越具体、场景越细节，AIGC 检测率越低
- 每章至少有 2-3 处具体场景/数据的叙述

**规则7：不完美允许**

- 允许偶尔的口语化连接词："说到这里"、"换个角度看"、"有意思的是"、"坦率地讲"
- 允许段落长度不均匀：有的段落可以 50 字（点睛一笔），有的段落可以 300+ 字（深入展开）
- 允许某些论点篇幅长、某些论点篇幅短——不要为了对称而硬凑
- 允许承认不确定性："目前尚无法确定"、"这一解释仍有待验证"、"笔者的观察是否具有普遍性，需要更大样本的检验"

**AIGC审查 Execution Protocol**（每次产出论文段落后强制执行——针对"AIGC最容易暴露的特征"）：
在输出每章内容前，请逐项自查：
1. **困惑度检测**（对应规则1）：本段句长是否均匀分布在25-35字区间？（无短句≤15字且无长句≥40字→**立即重写**，插入至少1个断言短句和1个展开长句）
2. **突发性检测**（对应规则3）：相邻两段是否以相同的结构模式开头？（如连续3段都是"主题句→展开"→**重排其中2段的开头方式**）
3. **引述指纹检测**（对应规则2）：是否连续使用了相同的引述句式超过2次？（如连续3次"张三(2020)指出..."→**必须替换为其他句式**）
4. **声音缺失检测**（对应规则4）：整段是否完全看不到任何研究者主观标记？（无"笔者认为"/"本研究"/"在实地观察中"等→**注入1处**，位置首选段中转折或段尾收束）
5. **抽象度检测**（对应规则6）：本段是否全部由抽象论断组成而无任何具体场景？（无具体数据/案例/观察细节→**将至少1处抽象表述替换为具体实例**）
6. **完美度检测**（对应规则7）：所有陈述都是确定性的？（全部使用"证明了""表明了""确认了"，无一例外→**将1-2处改为谨慎表达**："在一定程度上""初步显示""这一解释仍有待验证"）

#### 阶段6：引用来源确认

逐条检查正文中每处引用是否在文献池中有对应入池记录。池外引用视为错误，回溯搜索补入。

```
    ↕
```

### 开题报告模式（O1）

#### 开题报告标准结构

```
一、选题依据（~3000字）
  1.1 研究背景
  1.2 研究问题的提出
  1.3 研究目的与意义
  1.4 国内外研究现状（简版文献综述）

  2.1 [核心概念1]研究现状
  2.2 [核心概念2]研究现状
  2.3 研究述评与本研究的切入点

三、研究设计（~3000字）
  3.1 研究问题与研究假设
  3.2 研究对象
  3.3 研究方法
  3.4 研究工具
  3.5 数据收集与分析方案

四、研究创新点与预期成果（~1000字）

五、研究计划与时间安排
  （表格：时间阶段 → 主要工作 → 预期产出）

六、参考文献
```

## 模块三：学术论文评审

### 触发条件
评审论文、找问题、给出修改建议。

### 评审工作流

#### 阶段1：论文结构化读取
- 基础信息（标题、作者、期刊、年份）
- 研究设计类型
- 报告规范类型（RCT→CONSORT，综述→PRISMA，观察性→STROBE，质性→COREQ，混合→MMR）
- 预注册状态

#### 阶段2：五角色多视角评审

**角色A — 主编（EIC）**：
- 论文与目标期刊契合度
- 学术伦理合规性
- 综合各评审员意见

**角色B — 方法学评审员**：
- 研究设计合理性
- 统计功效分析
- 测量工具信效度
- 混淆变量控制
- 效应量报告完整性

**角色C — 领域专家评审员**：
- 文献覆盖全面性和平衡性
- 理论框架选择与论证
- 研究新颖性（近3年对比）
- 与文献对话深度

**角色D — 统计顾问**：
- 统计方法正确性
- 多重比较校正
- 效应量解释
- 置信区间报告
- 图形呈现诚实性

**角色E — 反方辩护人**：
- 系统性挑战核心假设
- 提出替代解释
- 检验结论稳健性
- 批评必须有证据基础，建设性

#### 阶段3：报告规范合规检查

| 研究设计 | 规范 | 关键检查项 |
|---------|------|-----------|
| RCT | CONSORT 2010 | 随机化方法、分配隐藏、盲法、流程图 |
| 系统综述/Meta | PRISMA 2020 | 检索策略、筛选流程图、偏倚评估 |
| 观察性研究 | STROBE | 设计说明、偏倚处理、样本量、统计方法 |
| 质性研究 | COREQ | 研究者特征、数据收集方法、数据分析 |
| 混合方法 | MMR | 混合设计类型、整合方式 |
| 准实验 | TREND | 干预描述、分配方法、基线比较 |
| 个案研究 | CARE | 患者信息、干预措施、时间线 |

#### 阶段4：AIGC 风险审查（可选）

当评审对象可能由 AI 辅助写作时，额外检查以下维度：

| 检查维度 | 高风险信号 | 低风险信号 |
|---------|-----------|-----------|
| 句长分布 | 句长集中在25-35字，几乎没有≤15字短句 | 短句与长句交替，有明显的落差感 |
| 段落结构 | 每段都是"主题句→展开→总结"标准三段式 | 段落开头方式多样，有些段无总结句 |
| 引述模式 | 连续使用相同引述句式（如"张三(2020)指出…"连续出现） | 引述句式交替（直接/间接/批判性/整合式等） |
| 语言风格 | 全文语气均匀，无个人立场，"端水"式客观转述 | 有研究者声音，有判断和态度 |
| 具体性 | 论述停留在抽象层面，缺少具体场景和数据细节 | 有具体案例、原始数据引述、实地观察细节 |

如发现高风险信号≥3项，在评审报告中标注"AIGC 辅助痕迹明显，建议人工润色后重新审视"。

#### 阶段5：综合评分（0-100分）

| 维度 | 权重 | 评分 |
|------|------|------|
| 研究问题与设计 | 20% | /100 |
| 方法论严谨性 | 25% | /100 |
| 结果呈现 | 15% | /100 |
| 讨论与解释 | 20% | /100 |
| 写作与格式 | 10% | /100 |
| 学术贡献 | 10% | /100 |
| **总分** | **100%** | **/100** |

#### 阶段6：综合评审报告

1. 分维度评分表 + 总分
2. 五角色独立评审意见
3. 报告规范合规结果
4. 优先修改建议 Top 5（按严重程度排序）

### 输出文件
`评审报告_<论文标题>_<日期>.md`

## 模块四：学术流程编排（国内硕士培养周期）

### 适用场景
管理硕士论文从开题到答辩的全过程。

### 培养阶段管理

| 阶段 | 名称 | 核心任务 | 输出物 | 检查点 |
|------|------|---------|--------|--------|
| 1 | 选题与开题 | 文献调研→确定选题→撰写开题报告→开题答辩 | 开题报告 | ✅ 研究问题可回答？方法可行？ |
| 2 | 文献综述 | 模块一全流程（轻量模式为主） | 综述+文献池 | ✅ 文献池足够支撑论文？ |
| 3 | 研究设计 | 设计类型→工具→样本方案→伦理审批 | 设计文档 | ✅ 工具有信效度？伦理通过？ |
| 4 | 数据收集 | 执行/获取数据，质控 | 原始数据+编码本 | ✅ 数据质量合格？ |
| 5 | 数据分析 | 描述→推断→效应量→可视化 | 分析报告 | ✅ 分析方案与设计一致？ |
| 6 | 论文写作 | 模块二学位论文模式全流程 | 初稿 | ✅ 所有引用来自文献池？ |
| 7 | 中期检查 | 汇报研究进展，接受导师组检查 | 中期报告 | ✅ 进度与计划匹配？ |
| 8 | 论文修改 | 导师意见修改 → 定稿 | 定稿 | ✅ 导师意见全部回应？ |
| 9 | 预答辩 | 预答辩汇报，收集意见 | 预答辩反馈 | ✅ 关键问题已解决？ |
| 10 | 正式答辩 | 答辩PPT + 答辩陈述 + 回答问题 | 答辩通过 | ✅ 答辩材料完整？ |

**阶段7（中期检查）的位置**：通常在阶段4-5之间或阶段6初稿完成后，具体以学校培养方案为准。可与用户确认。

### 伦理审查指南（A5）

#### 伦理审查在流程中的位置

伦理审查通常在研究设计完成后、数据收集前进行。时间线：

```

```

#### 伦理申请表核心内容

```
一、研究基本信息
  ├── 研究题目
  ├── 研究者姓名、单位、联系方式
  ├── 导师姓名、联系方式
  ├── 研究起止时间
  └── 资金来源

二、研究概述（500-800字）
  ├── 研究目的
  ├── 研究问题
  ├── 研究对象（人数、来源、入选标准）
  ├── 研究方法概述
  └── 数据收集方式

三、风险与收益评估
  ├── 本研究的风险：
  │   ├── 心理风险（如：讨论敏感话题是否引起不适）
  │   ├── 隐私风险（如：数据泄露可能性）
  │   └── 时间成本（如：占用参与者多少时间）
  ├── 风险控制措施
  └── 研究对参与者的直接/间接收益

四、知情同意方案
  ├── 知情同意书（见下方模板）
  ├── 参与者如何获取知情同意
  ├── 特殊群体的额外保护（如：未成年人需监护人同意）
  └── 参与者有权随时退出且不受影响

五、数据管理方案
  ├── 数据存储方式与地点
  ├── 数据保存期限
  ├── 数据匿名化措施
  ├── 谁有权访问数据
  └── 研究结束后的数据处理

六、研究工具（附）
  ├── 问卷/量表全文
  ├── 访谈提纲
  └── 观察记录表

七、研究者承诺
  └── 签名+日期
```

## 附录A：实用写作 Prompt 模板

> 💡 以下模板基于 Role/Task/Constraints/Execution Protocol 格式设计。每个模板末尾有自检环节。直接复制到对话中使用。

### A.1 去AI味（中文教育学论文）

````markdown
# Role
**【身份定位】** 你是一位教育学领域资深编辑。
**【领域锚定】** 你深谙《教育研究》《中国特殊教育》《学前教育研究》等核心期刊的审稿标准与写作规范，熟悉国内硕士论文从开题到答辩的全流程要求。
**【品味声明】** 你对"AI味"有极高的敏感度——那些看似华丽实则空洞的渲染表达、机械的连接词堆砌、千篇一律的三段式段落结构，在你眼中无处遁形。你的专长不是"换词"，而是理解原文意图后，用一位真正的教育学研究者会用的方式重新表达。你秉持**宁缺毋滥**的原则：如果原文已经好，你会坦然地说"不用改"。

# Task
请对我提供的【中文教育学论文段落】进行"去AI化"重写，使其语言风格严谨、客观、有研究者个人声音。

# Constraints
1. **修改阈值（⚠️ 核心原则——最高优先级）**：
   - **必须修改**：检测到以下 AI 特征时才动手——无实质信息量的渲染表达、机械连接词堆砌、英式长定语、"首先其次最后"罗列、"端水"式客观转述
   - **禁止修改**：如果原文已经自然、严谨、无明显 AI 痕迹 → **保留原样，不要为了证明自己有用而强行修改**
   - **正向反馈要求**：当判定为"不修改"时，必须在 Part 2 中给出正式的肯定评价（见输出格式）

2. **词汇净化（核心任务）**：
   - 删除无实质信息量的渲染表达："毋庸置疑""不可磨灭的贡献""范式转移""颠覆性""深度赋能"
   - 替换 AI 高频词为领域自然表达（见下方词汇参考表）
   - **保持核心专业术语准确性，绝对不为了"去 AI 味"而随意替换领域内专有名词**

3. **句式与结构自然化（去翻译腔与机械感）**：
   - 消除英式长定语："一个...的...的..."→拆为短句或多个分句
   - 减少"被"字句→优先无主语句或主动表达
   - 拒绝机械列表过渡→通过句意因果自然衔接
   - **灵活处理**：若列举在当前语境下逻辑更清晰（如陈述步骤），可酌情保留

4. **研究者声音注入**：
   - 适当注入："笔者认为""在实地观察中""这一发现与XX的研究形成对话"
   - 对文献有判断态度——允许商榷、允许遗憾、允许承认不确定性
   - **但不过度**：每段 1-2 处即可，不必每段都有

5. **排版纯净（Word 适配）**：
   - 纯文本输出，严禁 Markdown 加粗/斜体/标题符号
   - 全文使用中文全角标点
   - 数学符号/英文术语周围保留合理空格

6. **输出格式（条件分支）**：
   - **Part 1 [正文]**：
     * 如果进行了重写 → 输出重写后的纯文本
     * 如果未修改（原文已好） → **原样完整输出原文**
   - **Part 2 [修改日志 / 质量判定]**：
     * 如果进行了修改 → 简要列举改了哪些 AI 特征（如"删除3处渲染表达""拆分2处长定语""注入1处研究者判断"）
     * 如果未修改 → 必须输出以下正式评价文本：
       **"[检测通过] 原文表达严谨自然，无明显 AI 痕迹，建议保留。"**
   - **围栏**：除以上两部分外，不要输出任何多余的对话、解释或前缀说明。

# Execution Protocol（定制化自检——针对"去AI味最容易犯的错误"）
在输出前，请逐项自查：
1. **拟人度检查**：闭上眼读一遍 Part 1，这像不像一位你认识的教育学学者写的论文？还是像一篇"被润色过"的文章？
2. **必要性核查**：我是否为了换词而换词？（如果是 → **撤销所有修改**，将 Part 1 还原为原文，Part 2 输出"[检测通过]"）
3. **过度处理检测**：我是不是把一些本来正常的学术表达也当成"AI味"给改了？比如"显著"这个词本身没有问题，只有"显著提升+没有任何数据支撑"才是问题
````

## 附录C：科学数据分析与学术图表生成

> ⚠️ **核心原则：先分析，后出表。** 学术论文中的数据工作不是"把数字画成图"，而是"经过完整的统计推断流程后，用图表呈现分析结果"。以下每一节都遵循这条原则：检验假设 → 选择方法 → 执行分析 → 效应量+置信区间 → 最后才出表出图。

### C.0 数据分析全流程（强制执行顺序）

当用户提供原始数据或结果数据时，必须按以下流程推进，**不可跳过任何一步**：

```
步骤1: 数据审查
  ├── 检查变量类型（连续/分类/等级）
  ├── 检查缺失值与异常值 → 报告处理方式
  └── 确认样本量与分析方法的匹配性（如回归分析要求样本量>自变量数×10）

步骤2: 描述统计
  ├── 连续变量：M(SD) / 中位数(IQR) + 偏度 + 峰度
  ├── 分类变量：频数 + 百分比
  └── 按分组变量分别报告

步骤3: 检验前提假设（决定后续方法选择的关键步骤）
  ├── 正态性 → Shapiro-Wilk（n<50）或 Kolmogorov-Smirnov（n≥50）
  ├── 方差齐性 → Levene's test 或 Bartlett's test
  ├── 球形假设 → Mauchly's test（重复测量方差分析时）
  ├── 线性关系 → 散点图 + Pearson r（回归分析前提）
  └── 多重共线性 → VIF（回归分析前提）

步骤4: 基于前提假设选择正确方法（见 C.2 方法选择决策树）
  ├── 参数检验（满足前提）→ t检验 / 方差分析 / Pearson相关 / 线性回归
  └── 非参数检验（不满足前提）→ Mann-Whitney U / Kruskal-Wallis / Spearman相关

步骤5: 执行假设检验
  ├── 报告检验统计量（t/F/χ²/U）、自由度、p值
  ├── 报告效应量（Cohen's d / η² / r / Cramer's V）
  └── 报告置信区间（均值差/相关系数/回归系数的95%CI）

步骤6: 多重比较校正（如适用）
  ├── 两两比较 → Bonferroni / Tukey HSD
  └── 标注校正后的显著性水平

步骤7: 基于分析结果生成图表
  ├── 表格：严格三线表格式，含检验统计量+效应量+显著性标记
  └── 图形：出版级分辨率（≥300dpi），含误差线/置信区间
```

### C.1 前提假设检验——方法选择的分水岭

> ⚠️ 这是学术论文中最容易被忽略但最关键的环节。方法选错，后续所有分析都站不住脚。

```python
import numpy as np
from scipy import stats

def test_normality(data, alpha=0.05):
    """正态性检验"""
    n = len(data)
    # n<50 用 Shapiro-Wilk；n>=50 用 Kolmogorov-Smirnov
    if n < 50:
        stat, p = stats.shapiro(data)
        method = 'Shapiro-Wilk'
    else:
        stat, p = stats.kstest(data, 'norm', args=(np.mean(data), np.std(data, ddof=1)))
        method = 'Kolmogorov-Smirnov'
    
    result = '正态分布' if p > alpha else '非正态分布'
    return method, stat, p, result

def test_homogeneity(*groups, alpha=0.05):
    """方差齐性检验（Levene's test）"""
    stat, p = stats.levene(*groups)
    result = '方差齐性' if p > alpha else '方差不齐'
    return stat, p, result

# 论文中报告格式：
# "经Shapiro-Wilk检验，实验组数据符合正态分布（W=0.97, p=0.42）；
#  Levene's检验表明三组方差齐性（F=1.23, p=0.31），满足单因素方差分析前提。"
```

### C.2 方法选择决策树

基于步骤3的前提假设结果，选择正确的统计方法：

```
                   ┌─ 满足正态+方差齐 → 独立样本t检验（两组）/ 单因素ANOVA（多组）
                   │
比较组间均值差异 ──┤
（连续因变量）      │                 ┌─ 方差齐 → Welch's t检验 / Welch ANOVA
                   │                 │
                   └─ 不满足正态 ──→ 检查方差 ─┤
                                     │
                                     └─ 方差不齐 → Mann-Whitney U / Kruskal-Wallis

                   ┌─ 满足正态 → Pearson相关系数
                   │
分析两变量关联 ────┤
                   │
                   └─ 不满足正态 → Spearman等级相关

                   ┌─ 满足正态+线性+方差齐+无多重共线 → 多元线性回归
预测因变量 ────────┤
                   └─ 不满足 → 先尝试变量变换（log/平方根），仍不满足则用非参数替代

                   ┌─ 前测无差异 → 重复测量ANOVA（满足球形假设）
                   │            → 混合设计ANOVA（不满足球形，用Greenhouse-Geisser校正）
前后测比较 ────────┤
                   └─ 前测有差异 → ANCOVA（协方差分析，以前测为协变量）
```

**论文中报告方法选择理由的范例**：

> "Kolmogorov-Smirnov检验表明三组数据均不满足正态分布（p<0.05），故采用Kruskal-Wallis H检验比较三组干预效果差异。事后两两比较采用Mann-Whitney U检验，经Bonferroni校正后显著性水平设为p<0.017（0.05/3）。"

### C.3 各检验方法及论文报告规范

#### C.3.1 独立样本t检验

```python
from scipy import stats

# 前提：已通过正态性和方差齐性检验
group_a = df[df['组别']=='实验组']['后测成绩']
group_b = df[df['组别']=='对照组']['后测成绩']

# Welch's t检验（不假设方差齐性，更稳健）
t, p = stats.ttest_ind(group_a, group_b, equal_var=False)

# 计算效应量 Cohen's d
d = (np.mean(group_a) - np.mean(group_b)) / np.sqrt(
    ((len(group_a)-1)*np.var(group_a, ddof=1) + (len(group_b)-1)*np.var(group_b, ddof=1))
    / (len(group_a)+len(group_b)-2)
)

# 均值差95%CI
se = np.sqrt(np.var(group_a,ddof=1)/len(group_a) + np.var(group_b,ddof=1)/len(group_b))
ci_low = (np.mean(group_a)-np.mean(group_b)) - 1.96*se
ci_high = (np.mean(group_a)-np.mean(group_b)) + 1.96*se
```

**论文表格格式**（数据分析后输出）：

| 变量 | 实验组(n=30) M±SD | 对照组(n=30) M±SD | t | df | p | Cohen's d | 95%CI |
|------|-------------------|-------------------|---|---|---|----------|-------|
| 后测成绩 | 85.2±8.3 | 78.1±9.6 | 3.07 | 56.3 | .003 | 0.79 | [2.5, 11.7] |

⚠️ 论文中不只要p值。必须同时报告：**检验统计量 + 自由度 + 效应量 + 置信区间**。

#### C.3.2 Mann-Whitney U检验（非参数替代）

```python
# 前提：不满足正态性假设
u, p = stats.mannwhitneyu(group_a, group_b, alternative='two-sided')

# 效应量 r = Z / sqrt(N)
z = stats.norm.ppf(1 - p/2)  # 如果p为双尾
r = z / np.sqrt(len(group_a) + len(group_b))
```

**论文报告**：`"Mann-Whitney U检验显示实验组成绩显著高于对照组，U=612.5, p=0.008, r=0.34。"`

#### C.3.3 单因素方差分析（ANOVA）

```python
# 前提：各组正态 + 方差齐性（Levene's p>0.05）
groups = [df[df['组别']==g]['后测成绩'] for g in ['对照组','实验组A','实验组B']]
f, p_anova = stats.f_oneway(*groups)

# 效应量 η²
# 计算组间与组内平方和
grand_mean = np.mean(df['后测成绩'])
ss_between = sum(len(g)*((np.mean(g)-grand_mean)**2) for g in groups)
ss_total = sum((x-grand_mean)**2 for x in df['后测成绩'])
eta_sq = ss_between / ss_total

# 事后两两比较：Tukey HSD
```

| **坐标轴** | 有标签+单位；刻度线朝内；顶部右侧边框隐藏 | `tick_params(direction='in')` + `spines` |
| **显著性** | `* p<.05, ** p<.01, *** p<.001` | 用 `ax.annotate()` 或 `ax.text()` 手动标注 |
| **表题/图题** | 表题在上方、图题在下方；正文中单独标注 | 不写入图片中 |
| **颜色** | 彩色和黑白打印均能区分 | 避免仅靠颜色区分（加标记形状、线型） |
